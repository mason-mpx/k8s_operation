## 二、各存储目录详细说明

### 2.1 制品存储 `storage/artifacts/`

| 属性 | 说明 |
|------|------|
| **用途** | 存储 CI/CD 流水线构建产出的制品文件（JAR、二进制、tar.gz 等） |
| **写入来源** | Jenkins 构建完成后通过 `/api/v1/cicd/artifact/upload` 接口上传，或用户手动上传 |
| **路径规则** | `storage/artifacts/{pipeline_id}/{YYYYMMDD}/{filename}` |
| **手动上传** | `storage/artifacts/manual/{YYYYMMDD}/{filename}` |
| **代码定义** | [`internal/app/services/cicd_artifact.go`](../internal/app/services/cicd_artifact.go) 常量 `ArtifactUploadDir` |
| **文件类型** | `.jar` / `.jar.gz`（Java）、二进制 `.gz`（Go）、`.tar.gz`（前端/Python） |
| **清理策略** | 通过 API 删除制品时同步删除文件（`os.Remove`），支持批量删除 |
| **Git 追踪** | **未被 .gitignore 忽略**（建议添加 `storage/artifacts/*` 到 .gitignore） |
| **安全校验** | 上传时计算 SHA256 哈希并存入数据库，下载时可校验完整性 |

**当前实际文件示例**：
```
storage/artifacts/
└── 4/                                    # pipeline_id=4
    ├── 20260426/
    │   └── springboot-hello-1.0.0.jar      # 17.0 MB（原始 JAR）
    └── 20260429/
        └── springboot-hello-1.0.0.jar.gz   # 17.2 MB（gzip 压缩后）
```

**核心代码路径**：

```
写入链路：
  Jenkins 构建完成
    → curl POST /api/v1/cicd/artifact/upload (multipart/form-data)
    → artifact_controller.go ArtifactUpload()
    → cicd_artifact.go (service) ArtifactUpload()
        → os.MkdirAll(storage/artifacts/{pid}/{date})
        → os.Create(storePath)
        → io.CopyBuffer(dst, file, 4MB_buffer)  ← 大缓冲区加速
        → SHA256 计算
        → 写入数据库 cicd_artifact 表

下载链路：
  前端点击下载
    → GET /api/v1/cicd/artifact/download/:id
    → artifact_controller.go ArtifactDownload()
    → os.Stat(filePath) 校验文件存在
    → ctx.File(artifact.FilePath)
```

### 2.2 应用日志 `storage/logs/`

| 属性 | 说明 |
|------|------|
| **用途** | 存储平台运行时日志 |
| **代码定义** | [`initialize/logger.go`](../initialize/logger.go) 中初始化 |
| **配置来源** | `configs/config.yaml` → `App.LogFileName` / `App.BusinessLogFileName` |
| **Git 追踪** | 已忽略（`.gitignore` 中 `storage/logs/*`） |

**日志文件明细**：

| 文件 | 类型 | 级别 | 格式 | 用途 |
|------|------|------|------|------|
| `app.log` | 系统主日志 | 由配置决定（默认 debug） | JSON + Console | 全局请求/错误/中间件日志 |
| `biz.log` | 业务日志 | Info | JSON | 独立业务操作记录（CRUD、部署等） |
| `ai.log` | AI 助手日志 | Debug | JSON + Console | 大模型请求/响应/Token 用量 |
| `app-*.log.gz` | 历史归档 | — | 压缩 | lumberjack 自动轮转压缩 |

**日志轮转配置**（`config.yaml`）：
```yaml
App:
  LogMaxSize: 1        # 单文件最大 1MB 后轮转
  LogMaxBackup: 3      # 保留最近 3 个历史文件
  LogMaxAge: 30        # 历史文件最长保留 30 天
  LogCompress: true    # 历史文件 gzip 压缩
```

**当前实际文件**：
```
storage/logs/
├── app.log                                 # 943.2 KB（当前系统日志）
├── biz.log                                 # 18.2 KB（业务日志）
├── ai.log                                  # 93.4 KB（AI 助手日志）
├── app-2026-04-26T09-28-17.032.log.gz      # 40.9 KB（历史归档）
├── app-2026-04-28T08-09-10.909.log.gz      # 50.2 KB（历史归档）
└── app-2026-04-28T16-21-01.163.log.gz      # 36.0 KB（历史归档）
```

### 2.3 配置文件 `configs/`

| 文件 | 用途 | 敏感 | Git 追踪 |
|------|------|------|----------|
| `config.yaml` | 主配置（数据库/Redis/Jenkins/AI 等） | **是**（含密码、API Key） | **已 gitignore** |
| `config.yaml.example` | 配置模板（脱敏示例） | 否 | 已提交 |
| `k8s.yaml` | K8s kubeconfig 集群认证 | **是**（含证书/Token） | **已 gitignore** |
| `k8s.yaml.example` | kubeconfig 模板 | 否 | 已提交 |
| `jenkins-templates/*.groovy` | Jenkins 流水线模板（4 种语言） | 否 | 已提交 |
| `dockerfile-templates/Dockerfile.*` | Dockerfile 构建模板（4 种语言） | 否 | 已提交 |

**配置文件读取链路**：
```
程序启动
  → cmd/k8soperation/main.go
  → initialize/setting.go  读取 config.yaml（优先环境变量 APP_CONFIG）
  → initialize/k8s_cluster_init.go  读取 k8s.yaml（优先环境变量 K8S_CONFIG）
  → Jenkins 流水线创建时，读取 configs/jenkins-templates/ 下对应模板
```

## 三、.gitignore 规则与安全建议

**当前 `.gitignore` 已忽略**：
```gitignore
bin/                    # 编译产物
*.exe                   # 可执行文件
*.log                   # 日志文件
configs/config.yaml     # 敏感主配置
configs/k8s.yaml        # 敏感 K8s 配置
storage/logs/*          # 日志目录
```

**建议补充忽略**：
```gitignore
# 制品文件（体积大，不应提交到 Git）
storage/artifacts/*
!storage/artifacts/.gitkeep
```

> **注意**：当前 `storage/artifacts/` 目录 **未被 .gitignore 忽略**，里面的 JAR/gz 文件（数十 MB）可能会被意外提交到 Git 仓库，建议尽快添加忽略规则。

## 四、存储路径与代码映射汇总

| 本地路径 | 类型 | 代码入口 | 配置项 |
|----------|------|----------|--------|
| `storage/artifacts/{pid}/{date}/` | 制品文件 | `services/cicd_artifact.go` → `ArtifactUploadDir` | 硬编码常量 |
| `storage/logs/app.log` | 系统日志 | `initialize/logger.go` → `SetupLogger()` | `App.LogFileName` |
| `storage/logs/biz.log` | 业务日志 | `initialize/logger.go` → `SetupLogger()` | `App.BusinessLogFileName` |
| `storage/logs/ai.log` | AI 日志 | `initialize/logger.go` → L86 | 硬编码 `"storage/logs/ai.log"` |
| `configs/config.yaml` | 主配置 | `initialize/setting.go` | 环境变量 `APP_CONFIG` |
| `configs/k8s.yaml` | K8s 配置 | `initialize/k8s_cluster_init.go` | 环境变量 `K8S_CONFIG` |
| `configs/jenkins-templates/*.groovy` | 流水线模板 | `models/cicd_pipeline.go` → `LanguageTemplateMap` | 硬编码映射表 |
| `configs/dockerfile-templates/Dockerfile.*` | 构建模板 | 平台页面选择使用 | — |
| `bin/k8soperation.exe` | 编译产物 | `Makefile` → `make build` | — |

## 五、磁盘空间估算

| 目录 | 当前大小 | 增长趋势 | 清理建议 |
|------|----------|----------|----------|
| `storage/artifacts/` | ~34 MB | 每次构建 +10~50 MB | 定期清理旧版本制品，或通过 API 批量删除 |
| `storage/logs/` | ~1.2 MB | 每天 ~0.5~2 MB | lumberjack 自动轮转，保留 3 份 × 30 天 |
| `bin/` | ~109 MB | 固定（每次覆盖） | 无需清理 |
| `configs/` | ~100 KB | 基本不变 | 无需清理 |
