# 流水线与发布管理模块说明

## 一、模块概述

| 模块 | 职责 | 后端路由 | 数据来源 |
|------|------|---------|---------|
| **流水线管理** | CI（持续集成）- 代码构建、测试、打镜像 | `/api/v1/k8s/cicd/pipeline/*` | 真实后端接口 |
| **发布管理** | CD（持续部署）- 镜像部署到 K8s 集群 | `/api/v1/k8s/cicd/release/*` | 真实后端接口 |
| **流水线模板** | 预定义流水线配置模板 | 未实现 | 前端 Mock 数据 |

---

## 二、核心区别

| 维度 | 流水线（Pipeline） | 发布单（Release） |
|-----|-------------------|------------------|
| **本质** | 配置模板 - 定义"怎么做" | 执行实例 - 记录"做了什么" |
| **生命周期** | 长期存在，可反复使用 | 一次性，执行完即结束 |
| **包含内容** | Git仓库、分支、Jenkins Job、环境变量 | 镜像版本、目标集群、部署状态 |
| **核心动作** | 触发构建（CI） | 执行部署（CD） |

### 简单类比

```
流水线 = 菜谱（记录怎么做这道菜）
发布单 = 订单（今天按这个菜谱做了一份）
```

## 三、使用模式

### 3.1 配合使用模式（CI + CD 联动）

**完整 DevOps 流程，推荐使用**

```
┌──────────────────────────────────────────────────────────────────────────┐
│  1. 创建流水线（配置 auto_deploy=true）                                    │
│     └─ Git仓库 + Jenkins Job + 部署目标配置                                │
└────────────────────────┬─────────────────────────────────────────────────┘
                         ↓
┌──────────────────────────────────────────────────────────────────────────┐
│  2. 运行流水线 → 触发 Jenkins 构建                                        │
│     └─ 拉代码 → 编译 → 测试 → 构建镜像 → 推送 Harbor                      │
└────────────────────────┬─────────────────────────────────────────────────┘
                         ↓
┌──────────────────────────────────────────────────────────────────────────┐
│  3. Jenkins 回调平台（构建完成）                                           │
│     └─ POST /api/v1/k8s/cicd/pipeline/callback                           │
│     └─ 携带: image, image_digest, status                                 │
└────────────────────────┬─────────────────────────────────────────────────┘
                         ↓
┌──────────────────────────────────────────────────────────────────────────┐
│  4. 平台自动创建发布单（如果 auto_deploy=true）                            │
│     └─ 根据流水线配置的 target_cluster_id, target_namespace 等           │
└────────────────────────┬─────────────────────────────────────────────────┘
                         ↓
┌──────────────────────────────────────────────────────────────────────────┐
│  5. Worker 执行部署                                                       │
│     └─ 更新 K8s Deployment → 等待 Pod Ready → 标记成功/失败              │
└──────────────────────────────────────────────────────────────────────────┘
```

#### 流水线配置示例（联动模式）

```json
// POST /api/v1/k8s/cicd/pipeline/create
{
  "name": "frontend-pipeline",
  "git_repo": "https://github.com/xxx/frontend.git",
  "git_branch": "main",
  "jenkins_job": "frontend-build",
  
  // 开启自动部署
  "auto_deploy": true,
  "target_cluster_id": 1,
  "target_namespace": "production",
  "target_workload_kind": "Deployment",
  "target_workload_name": "frontend-app",
  "target_container": "frontend",
  
  // 是否需要审批
  "require_approval": true
}
```

### 3.2 单独使用模式

#### 模式 A：只用流水线（纯 CI）

**场景**：只做代码构建、测试、打镜像，不自动部署

**适用情况**：
- 公共库/SDK 构建
- 多环境需要人工审核后才部署
- 构建产物需要人工验证

```json
// 创建流水线（auto_deploy=false）
{
  "name": "build-only-pipeline",
  "git_repo": "https://github.com/xxx/lib.git",
  "jenkins_job": "lib-build",
  "auto_deploy": false  // 不自动部署
}
```

**结果**：Jenkins 构建完成后，镜像推到 Harbor，流程结束。部署由人工决定。

---

#### 模式 B：只用发布单（纯 CD）

**场景**：镜像已存在（手动打包/第三方 CI/外部镜像），只需要部署到 K8s

**适用情况**：
- 使用第三方镜像（如官方 nginx、redis）
- 镜像由其他 CI 系统构建（GitHub Actions、GitLab CI）
- 紧急发布/回滚到指定版本

```json
// POST /api/v1/k8s/cicd/release/create
{
  "app_name": "my-app",
  "namespace": "production",
  "workload_kind": "Deployment",
  "workload_name": "my-app-deployment",
  "container_name": "main",
  
  // 直接指定镜像（不依赖流水线构建）
  "image_repo": "harbor.example.com/project/my-app",
  "image_tag": "v1.2.3",
  
  // 目标集群（支持多集群并行部署）
  "cluster_ids": [1, 2, 3]
}
```

**结果**：Worker 直接拿镜像更新 K8s Deployment，不涉及 Jenkins 构建。

## 四、场景对照表

| 场景 | 流水线 | 发布单 | 适用情况 |
|-----|-------|-------|---------|
| **CI+CD 联动** | 构建镜像 + 触发部署 | 自动创建 | 标准 DevOps 流程 |
| **纯 CI** | 只构建镜像 | 不创建 | 公共库/SDK 构建 |
| **纯 CD** | 不使用 | 手动创建 | 第三方镜像/紧急发布 |
| **多集群发布** | 构建一次 | 多任务并行 | 镜像同步到多个集群 |
| **版本回滚** | 不使用 | 指定旧版本镜像 | 快速回滚 |

| POST | `/api/v1/k8s/cicd/pipeline/run` | 运行流水线（触发 Jenkins 构建） |
| POST | `/api/v1/k8s/cicd/pipeline/stop` | 停止流水线 |
| GET | `/api/v1/k8s/cicd/pipeline/logs` | 获取构建日志 |
| GET | `/api/v1/k8s/cicd/pipeline/status` | 获取实时状态 |
| GET | `/api/v1/k8s/cicd/pipeline/stages` | 获取阶段数据 |
| GET | `/api/v1/k8s/cicd/pipeline/history` | 获取运行历史 |

### 5.2 发布管理 API

| 方法 | 路径 | 说明 |
|-----|------|------|
| GET | `/api/v1/k8s/cicd/release/list` | 获取发布单列表 |
| GET | `/api/v1/k8s/cicd/release/detail` | 获取发布单详情 |

## 六、数据模型关系

```sql
-- 流水线（CI 配置）
cicd_pipeline
├── id
├── name, git_repo, jenkins_job
├── auto_deploy, target_cluster_id...  -- 部署配置
└── last_run_id → cicd_pipeline_run

-- 流水线运行记录
cicd_pipeline_run  
├── pipeline_id → cicd_pipeline
├── build_number, status
└── image_url, image_digest  -- 构建产物

-- 发布单（CD 执行）
cicd_release
├── id
├── app_name, namespace, workload_name
├── image_repo, image_tag  -- 可来自流水线或手动指定
└── status (Pending/Queued/Running/Succeeded/Failed/Canceled)

-- 发布任务（多集群并行）
cicd_release_task
├── release_id → cicd_release
├── cluster_id
├── target_image, prev_image  -- 用于回滚
└── status
```

## 七、状态流转

### 7.1 流水线运行状态

```
idle → running → success/failed
         ↓
       stopped（手动停止）
```

### 7.2 发布单状态

```
Pending → Queued → Running → Succeeded
                      ↓
                   Failed → (Retry) → Queued
                      ↓
                   Canceled
                      ↓
                   Rollback（已部署的触发回滚）
```

## 八、最佳实践

### 8.1 标准项目（推荐）

1. 创建流水线，开启 `auto_deploy=true`
2. 配置审批流程 `require_approval=true`（生产环境）
3. 代码推送后自动触发构建和部署

### 8.2 多环境部署

1. 创建流水线，`auto_deploy=false`
2. Jenkins 构建成功后，手动选择目标环境创建发布单
3. 支持同一镜像部署到 dev/staging/production

### 8.3 紧急发布

1. 跳过流水线，直接创建发布单
2. 指定已验证的镜像版本
3. 快速部署到目标集群

### 8.4 快速回滚

1. 找到历史成功的发布单
2. 调用回滚接口，自动使用 `prev_image` 创建新发布单
3. Worker 自动执行回滚部署
