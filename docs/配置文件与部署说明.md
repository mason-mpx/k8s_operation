# 配置文件与部署说明

本文档说明 K8sOperation 在不同环境下的配置差异，特别是日志路径等关键配置项。

---

## 目录结构

```
k8s_operation/
├── configs/
│   ├── config.yaml           # 实际配置文件（本地开发用，已被 .gitignore 忽略）
│   ├── config.yaml.example   # 配置模板（提交到 Git）
│   └── k8s.yaml.example      # K8s 集群连接配置模板
├── storage/
│   └── logs/                 # 日志输出目录（本地运行时自动创建）
│       ├── app.log           # 系统日志
│       └── biz.log           # 业务日志
└── Dockerfile                # 容器构建文件
```

---

## 环境配置对比

| 配置项 | 本地开发 | K8s/Docker 部署 |
|--------|----------|-----------------|
| `LogFileName` | `storage/logs/app.log` | `/app/storage/logs/app.log` |
| `BusinessLogFileName` | `storage/logs/biz.log` | `/app/storage/logs/biz.log` |
| `GlobalKubeConfigPath` | `configs/k8s.yaml` | `/app/configs/k8s.yaml` |

### 路径说明

- **本地开发**：使用**相对路径**，相对于项目根目录
- **K8s/Docker 部署**：使用**绝对路径**，因为容器工作目录是 `/app`，Dockerfile 中已预先创建 `/app/storage/logs` 目录

## 本地开发配置

### 1. 复制配置模板

```bash
# 复制配置文件
cp configs/config.yaml.example configs/config.yaml
cp configs/k8s.yaml.example configs/k8s.yaml
```

### 2. 修改 config.yaml

确保日志路径使用**相对路径**：

```yaml
App:
  LogLevel: debug
  LogFileName: storage/logs/app.log         # 相对路径
  BusinessLogFileName: storage/logs/biz.log # 相对路径
  GlobalKubeConfigPath: configs/k8s.yaml
```

### 3. 启动服务

```bash
go run ./cmd/k8soperation/main.go
```

启动后会自动创建 `storage/logs/` 目录。

## Docker 部署配置

### Dockerfile 关键配置

```dockerfile
# 创建工作目录和日志目录
WORKDIR /app
RUN mkdir -p /app/configs /app/logs /app/storage/logs && \
    chown -R appuser:appgroup /app
```

### 构建镜像

```bash
docker build -t k8s-operation:latest .
```

### 运行容器

```bash
docker run -d \
  -p 8080:8080 \
  -v /path/to/config.yaml:/app/configs/config.yaml \
  -v /path/to/k8s.yaml:/app/configs/k8s.yaml \
  k8s-operation:latest
```

### 2. 创建 Deployment

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: k8s-operation
  namespace: k8s-platform
spec:
  replicas: 2
  selector:
    matchLabels:
      app: k8s-operation
  template:
    metadata:
      labels:
        app: k8s-operation
    spec:
      containers:
        - name: k8s-operation
          image: your-registry/k8s-operation:latest
          ports:
            - containerPort: 8080
          volumeMounts:
            - name: config-volume
              mountPath: /app/configs
            - name: log-volume
              mountPath: /app/storage/logs
          livenessProbe:
            httpGet:
              path: /health
              port: 8080
            initialDelaySeconds: 10
            periodSeconds: 30
          readinessProbe:
            httpGet:
              path: /health
              port: 8080
            initialDelaySeconds: 5
            periodSeconds: 10
          resources:
            requests:
              memory: "128Mi"
              cpu: "100m"
            limits:
              memory: "512Mi"
              cpu: "500m"
      volumes:
        - name: config-volume
          configMap:
            name: k8s-operation-config
        - name: log-volume
          emptyDir: {}
```

### 3. 创建 Service

```yaml
apiVersion: v1
kind: Service
metadata:
  name: k8s-operation-svc
  namespace: k8s-platform
spec:
  type: ClusterIP
  ports:
    - port: 8080
      targetPort: 8080
      protocol: TCP
  selector:
    app: k8s-operation
```

## 常见问题

### Q1: 本地运行报错 "open configs/config.yaml: no such file or directory"

**原因**：未创建配置文件

**解决**：
```bash
cp configs/config.yaml.example configs/config.yaml
```

### Q2: 本地运行时日志目录未创建

**原因**：`config.yaml` 中使用了绝对路径 `/app/storage/logs/...`

**解决**：修改为相对路径
```yaml
App:
  LogFileName: storage/logs/app.log
  BusinessLogFileName: storage/logs/biz.log
```

### Q3: Docker/K8s 部署时日志写入失败

**原因**：可能使用了相对路径

**解决**：确保使用绝对路径
```yaml
App:
  LogFileName: /app/storage/logs/app.log
  BusinessLogFileName: /app/storage/logs/biz.log
```

## 配置文件版本管理建议

1. **不要提交** `configs/config.yaml` 到 Git（已在 `.gitignore` 中忽略）
2. **保持更新** `configs/config.yaml.example` 作为模板
3. **K8s 部署**时通过 ConfigMap 或 Secret 管理敏感配置
4. **敏感信息**（密码、密钥等）建议使用 K8s Secret 而非 ConfigMap
