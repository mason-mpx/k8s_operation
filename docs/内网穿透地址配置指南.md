# 内网穿透地址配置指南

> 当花生壳等内网穿透工具的映射端口发生变化时，需要同步修改以下文件。

## 地址映射关系

| 映射名称 | 外网地址 | 内网地址 |
|---------|---------|---------|
| 前端 | `http://708iuyd54169.vicp.fun:<前端端口>` | `192.168.10.12:5173` |
| 后端 | `http://708iuyd54169.vicp.fun:<后端端口>` | `192.168.10.12:8080` |

## 需要修改的文件清单（共 4 个）

### 1. 后端配置 — `configs/config.yaml`

修改 Jenkins 回调和平台地址的端口：

```yaml
Jenkins:
  CallbackURL: "http://708iuyd54169.vicp.fun:<后端端口>"  # Jenkins 回调地址
  PlatformURL: "http://708iuyd54169.vicp.fun:<前端端口>"   # 钉钉通知跳转链接
```

### 2. 后端 CORS — `initialize/server.go`

修改 `AllowOrigins` 中的两条穿透地址：

```go
AllowOrigins: []string{
    "http://localhost:5173",
    "http://127.0.0.1:5173",
    // 内网穿透
    "http://708iuyd54169.vicp.fun:<前端端口>",  // 前端
    "http://708iuyd54169.vicp.fun:<后端端口>",  // 后端
    // ...
},
```

### 3. 前端环境变量 — `k8s-web/.env`

修改远程 API 端口（后端端口）：

```env
VITE_REMOTE_API_PORT=<后端端口>
```

### 4. 前端 HTTP 模块 — `k8s-web/src/api/http.js`

修改 fallback 默认端口（与 `.env` 保持一致）：

```js
const backendPort = import.meta.env.VITE_REMOTE_API_PORT || '<后端端口>'
```

## 无需修改的文件

| 文件 | 原因 |
|------|------|
| `k8s-web/vite.config.js` | `allowedHosts` 只配了域名，不含端口 |
| `k8s-web/.env.development` | 本地开发走 Vite proxy，不涉及穿透端口 |

## 修改后生效步骤

```bash
# 1. 重新编译并启动后端
make build && make run
# 或开发模式
make run-local

# 2. 重启前端 dev server
cd k8s-web
npm run dev
```
