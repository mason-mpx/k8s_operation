# 构建探针管理（Build Agent Management）

## 1. 功能概述

**构建探针管理** 是 K8sOperation 平台 CI/CD 模块下的核心功能，用于统一管理构建过程中需要注入到 Docker 镜像的 Agent 文件（如可观测性探针、诊断工具、安全扫描器等）。

### 核心价值

| 传统方式 | 平台管理方式 |
|---------|------------|
| Dockerfile 中 `RUN wget` 下载（国内网络不稳定） | 平台统一存储，流水线自动拉取 |
| 每个项目单独维护 Agent 版本 | 负责人统一维护，100+ 项目复用 |
| 升级需要修改所有项目 Dockerfile | 平台更新一次，所有构建自动生效 |
| 构建失败时难以定位是网络还是代码问题 | 离线可用，构建稳定性 99.9%+ |

### 设计特点

- **零改动接入**：上传 Agent 后，已有流水线自动生效，无需修改 Dockerfile 或代码
- **多级降级**：平台 API → 项目自带 → Maven 镜像下载，确保构建不中断
- **动态注入**：Dockerfile 在构建时自动生成，包含所有已启用探针的 COPY 和 ENV 配置
- **按语言隔离**：Java/Go/Python 各自维护独立的探针列表

## 3. 数据模型

### CicdBuildAgent 表结构

| 字段 | 类型 | 说明 |
|------|------|------|

| env_key | VARCHAR(100) | 注入的环境变量 Key |
| env_value | VARCHAR(2000) | 注入的环境变量 Value |
| status | VARCHAR(20) | 状态：`active` / `inactive` |
| download_count | INT | 下载次数 |
| used_count | INT | 构建引用次数 |
| created_by | BIGINT | 创建人 ID |

## 4. API 接口

### 基础路径：`/api/v1/k8s/cicd/agent`

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | `/list` | 获取探针列表（分页、筛选） | super_admin, platform_admin, cicd_admin |
| GET | `/detail?id=` | 获取探针详情 | 同上 |
| POST | `/upload` | 上传探针文件（multipart/form-data） | 同上 |
| POST | `/update` | 更新探针信息 | 同上 |
| POST | `/toggle` | 切换启用/停用状态 | 同上 |
| POST | `/delete` | 删除探针（含文件） | 同上 |
| GET | `/download?id=` / `?name=` | 下载探针文件（流水线使用） | 无鉴权（内部调用） |
| GET | `/by-scope?scope=java` | 获取指定语言已启用探针列表 | 无鉴权（流水线调用） |

### 上传请求示例

```bash
curl -X POST /api/v1/k8s/cicd/agent/upload \
  -F "file=@opentelemetry-javaagent-1.33.0.jar" \
  -F "name=opentelemetry-javaagent" \
  -F "display_name=OpenTelemetry Java Agent" \
  -F "category=observability" \
  -F "scope=java" \
  -F "version=1.33.0" \
  -F "docker_copy_dest=/app/opentelemetry-javaagent.jar" \
  -F "env_key=JAVA_TOOL_OPTIONS" \
  -F "env_value=-javaagent:/app/opentelemetry-javaagent.jar"
```

## 5. 流水线集成逻辑

### Stage: Prepare Build Agents

流水线 `java-spring-pipeline.groovy` 中的 `Prepare Build Agents` 阶段按以下策略执行：

```
优先级 1: 项目根目录自带 opentelemetry-javaagent.jar → 直接复用
优先级 2: 平台 API → GET /agent/by-scope?scope=java → 逐个下载所有已启用 Agent
优先级 3: 降级 → 阿里云 Maven → Maven Central → GitHub 下载 OTEL Agent
```

### 动态 Dockerfile 生成

当项目**没有自带 Dockerfile** 时，流水线自动生成包含所有探针的 Dockerfile：

```dockerfile
FROM registry.cn-hangzhou.aliyuncs.com/k8s-gos/java:17-jre-alpine
ENV TZ=Asia/Shanghai
WORKDIR /app
RUN addgroup -S appgroup && adduser -S -G appgroup appuser
RUN mkdir -p /app/logs && chown -R appuser:appgroup /app
# ↓↓↓ 以下 COPY 行由平台 Agent 列表动态生成 ↓↓↓
COPY .agents/opentelemetry-javaagent/opentelemetry-javaagent.jar /app/opentelemetry-javaagent.jar
COPY .agents/skywalking-agent/skywalking-agent.jar /app/skywalking-agent.jar
# ↑↑↑ 动态生成结束 ↑↑↑
COPY target/*.jar /app/app.jar
USER appuser
EXPOSE 8080
ENV JAVA_TOOL_OPTIONS="-javaagent:/app/opentelemetry-javaagent.jar"
ENV OTEL_OPTS="-javaagent:/app/opentelemetry-javaagent.jar -Dotel.service.name=java-app ..."
ENV JAVA_OPTS="-XX:MaxRAMPercentage=75.0 -XX:+UseG1GC ..."
ENTRYPOINT ["sh", "-c", "exec java $OTEL_OPTS $JAVA_OPTS -jar /app/app.jar"]
```

### 文件目录结构

```
workspace/
├── .agents/                              ← 流水线自动创建
│   ├── opentelemetry-javaagent/
│   │   └── opentelemetry-javaagent.jar   ← 从平台下载
│   └── skywalking-agent/
│       └── skywalking-agent.jar          ← 从平台下载
├── target/
│   └── app-1.0.0.jar                     ← Maven 编译产出
└── .Dockerfile.runtime                   ← 自动生成的 Dockerfile
```

## 6. 使用指南

### 6.1 添加新的 Agent（全自动模式）

**只需一步操作，无需修改任何代码或配置：**

1. 在前端 `/cicd/agents` 页面点击「上传探针」
2. 填写探针信息并上传文件
3. 确保状态为「已启用」

所有使用平台流水线的 Java 项目，下次构建时会自动包含新 Agent。

### 6.2 升级已有 Agent

1. 在探针列表中找到目标 Agent
2. 点击删除旧版本
3. 重新上传新版本文件（保持 name 不变）

或者直接在编辑弹窗中修改版本号和文件。

### 6.3 临时停用 Agent

点击探针卡片上的「暂停」按钮，Agent 状态变为 `inactive`，流水线构建时不会拉取该 Agent。

### 6.4 项目自定义 Agent

如果某个项目需要特殊版本的 Agent，直接在项目根目录放置 `opentelemetry-javaagent.jar`，流水线会优先使用项目自带版本。

## 9. 安全与权限

- **前端访问**：仅 `super_admin`、`platform_admin`、`cicd_admin` 角色可访问探针管理页面
- **下载接口**：`/agent/download` 和 `/agent/by-scope` 不做鉴权（供 Jenkins 内网直接调用）
- **文件校验**：上传时计算 SHA256 并存储，下载时可通过响应头 `X-Agent-Sha256` 校验
- **存储隔离**：每个 Agent 独立目录，删除操作同步清理文件

## 10. 常见问题

### Q: 平台不可用时构建会失败吗？

不会。流水线有三级降级策略：
1. 平台 API 下载
2. 项目自带 Agent（根目录）
3. Maven 公共仓库下载

即使所有源都不可用，也仅创建空占位文件，OTEL 不生效但**不阻断构建**。

### Q: 如何确认 Agent 是否正确注入到镜像？

查看构建日志中 `[Agents]` 开头的输出：
```
[Agents] 平台返回 2 个已启用探针
[Agents] ✅ OpenTelemetry Java Agent v1.33.0 (18234567 bytes)
[Agents] ✅ SkyWalking Agent v9.1.0 (21456789 bytes)
[Agents] === 准备完成: opentelemetry-javaagent, skywalking-agent ===
[Build Image] 自动生成 Dockerfile（动态注入 2 个探针）
```

### Q: 支持非 Java 语言吗？

数据模型支持 `go`、`python`、`all` scope，但当前仅 `java-spring-pipeline.groovy` 实现了自动拉取逻辑。Go/Python 流水线可参照相同模式扩展。

### Q: 如何手动测试下载接口？

```bash
# 按名称下载
wget -O agent.jar "http://platform:8080/api/v1/k8s/cicd/agent/download?name=opentelemetry-javaagent"

# 查看指定语言的已启用探针
curl "http://platform:8080/api/v1/k8s/cicd/agent/by-scope?scope=java"
```
