# 前端流水线创建向导使用手册

> 本文档详细说明 K8sOperation 平台**前端流水线创建向导**的完整操作流程、表单字段说明、前后端接口对照及验证规则。

---

## 一、功能入口

| 路径 | 说明 |
|------|------|
| 左侧菜单 → **CI/CD** → **流水线管理** | 流水线列表页 |
| 点击右上角 **「创建流水线」** 按钮 | 进入创建向导 |
| 前端路由 | `/cicd/pipeline/create` |
| 编辑模式 | `/cicd/pipeline/:id/edit`（复用同一组件） |

**核心组件：** `k8s-web/src/views/cicd/PipelineCreate.vue`（3400+ 行）

---

## 二、向导步骤总览

创建向导采用 **5 步骤左侧导航 + 右侧表单** 布局，支持步骤间跳转和快速模板预填：

```
┌─────────────────────────────────────────────────┐
│  左侧步骤导航            │  右侧表单内容          │
│                          │                        │
│  ① 基本信息（名称和描述）│  ┌──────────────────┐  │
│  ② 代码仓库（Git 配置） │  │   当前步骤表单    │  │
│  ③ Jenkins（构建配置）   │  │                  │  │
│  ④ 部署策略（滚动更新）  │  │                  │  │
│  ⑤ 自动部署（K8s 目标）  │  └──────────────────┘  │
│                          │                        │
│  ┌───────────────────┐   │  [上一步]  步骤x/5  [下一步] │
│  │ 快速模板（下拉选择）│   │                        │
│  └───────────────────┘   │                        │
└─────────────────────────────────────────────────┘
```

## 三、各步骤详细说明

### 步骤 1：基本信息

| 字段 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| **流水线名称** | 文本输入 | ✅ | 使用小写字母、数字和连字符 | `frontend-deploy-pipeline` |
| **描述** | 多行文本 | ❌ | 简要描述此流水线用途 | `前端 Vue 项目自动构建部署` |

**验证规则：**
- 前端：`name` 不能为空（`trim()` 后判断）
- 后端：`name` 必填，长度 1-100 字符

---

### 步骤 2：代码仓库

| 字段 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| **Git 仓库地址** | URL 输入 | ✅ | HTTPS/SSH 格式的 Git 仓库 | `https://github.com/org/repo.git` |
| **获取分支** | 按钮 | - | 调用后端自动获取远程分支列表 | 点击后展示分支下拉 |
| **分支** | 下拉选择/文本输入 | ✅ | 已获取分支时下拉选择，否则手动输入 | `main` |

**交互细节：**
1. 输入仓库地址后，点击 **「获取分支」** 按钮 → 调用 `POST /api/v1/k8s/cicd/git/branches`
2. 成功后展示分支列表（支持搜索过滤），标记 `default` 分支
3. 点击分支项即选中，底部显示 "已选择：main"
4. 若未获取分支，退化为手动输入框

**验证规则：**
- 前端：`git_repo` 和 `git_branch` 不能为空
- 后端：`git_repo` 必填 + URL 格式校验

**提示卡片：** "确保仓库地址可访问，并已在 Jenkins 中配置好相应的凭证"

---

### 步骤 3：Jenkins 配置

| 字段 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| **Jenkins 服务器地址** | URL 输入 | ❌ | 留空使用系统默认 Jenkins | `http://jenkins.example.com:8080` |
| **Jenkins Job 名称** | 文本输入 | ✅（前端校验） | Jenkins 中已创建的 Job 名称 | `k8s-builder-java` |
| **环境变量** | 动态键值对 | ❌ | 可添加多组 `变量名=变量值` | `JAVA_VERSION=17` |

**环境变量操作：**
- 可折叠/展开，右侧显示变量数量标记
- 点击 **「添加环境变量」** 新增一行
- 每行支持独立删除

**验证规则：**
- 前端：`jenkins_job` 不能为空
- 后端：`jenkins_job` 不强制必填（模板化时可由 `language_type` 自动推导），`language_type` 校验值域 `go/java/frontend/python/custom`

> **模板化提示：** 如果使用快速模板，`jenkins_job` 会由平台根据 `language_type` 自动映射（如 `java` → `k8s-builder-java`），无需手动填写。

---

### 步骤 4：部署策略

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| **部署环境** | 芯片选择器 | ❌ | 4 个环境：开发/测试/预发/生产 |
| **服务类型** | 卡片选择器 | ❌ | Java / Go / Node.js / Python |
| **资源模板** | 下拉选择 | ❌ | 从预定义模板加载配置，或自定义 |
| **部署策略** | 卡片选择（4选1）| ❌ | 滚动更新/重新创建/蓝绿部署/金丝雀 |
| **副本数** | 数字控制器 | ❌ | 最小 1，最大 100，默认 3 |
| **资源配置** | 可折叠区域 | ❌ | Limits / Requests 的 CPU 和内存 |

#### 部署环境选项

| 环境 | 标识 | 颜色 | 说明 |
|------|------|------|------|
| 开发环境 | `dev` | 绿色 `#52c41a` | 用于开发调试 |
| 测试环境 | `test` | 蓝色 `#1890ff` | 用于单元测试 |
| 预发环境 | `staging` | 黄色 `#faad14` | 用于集成测试 |
| 生产环境 | `prod` | 红色 `#ff4d4f` | 需要审批流程 |

#### 部署策略选项

| 策略 | 标识 | 说明 |
|------|------|------|
| 滚动更新 | `rollingUpdate` | 逐步替换旧版本（**默认**） |
| 重新创建 | `recreate` | 停止后再启动 |
| 蓝绿部署 | `blueGreen` | 零停机切换 |
| 金丝雀 | `canary` | 渐进式发布 |

#### 资源配置默认值

| 类型 | CPU | 内存 |
|------|-----|------|
| Limits（上限） | `500m` | `512Mi` |
| Requests（请求） | `200m` | `256Mi` |

#### 资源校验功能

选择资源模板后会自动调用 `POST /api/v1/k8s/cicd/resource/validate` 进行校验：
- ✅ 校验通过：绿色提示
- ❌ 校验失败：红色错误列表
- ⚠️ 风险提示：显示风险等级（高风险/中风险）
- 🔒 审批提示：生产环境高配置需 SRE 角色审批

---

### 步骤 5：自动部署

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| **启用自动部署** | 开关 | ❌ | 构建成功后自动更新 K8s 工作负载 |
| **部署环境** | 卡片选择 | ✅* | 开启自动部署后必选 |
| **部署前审批** | 开关 | ❌ | 生产环境建议开启 |
| **目标集群** | 下拉选择 | ✅* | 从平台已注册的集群列表选择 |
| **目标命名空间** | 下拉选择 | ✅* | 选集群后自动加载命名空间列表 |
| **工作负载类型** | 卡片选择（3选1）| ✅* | Deployment / StatefulSet / DaemonSet |
| **工作负载名称** | 下拉选择/文本输入 | ✅* | 选命名空间后自动加载工作负载列表 |
| **容器名称** | 文本输入 | ❌ | 指定容器，留空更新第一个容器 |

> ✅* = 启用自动部署后必填

**级联加载逻辑：**
```
选择集群 → 加载命名空间列表
    ↓
选择命名空间 → 根据工作负载类型加载工作负载列表
    ↓
选择工作负载 → 配置摘要自动更新
```

**配置摘要面板：**  
显示最终部署路径：`集群 / 命名空间 / 工作负载类型 / 工作负载名称`  
显示部署环境 + 审批标记

## 四、快速模板功能

左侧导航下方的 **「快速模板」** 下拉框可一键预填表单：

1. 选择模板 → 调用 `GET /api/v1/k8s/cicd/template/list` 获取模板列表
2. 选中后自动填充：名称、描述、Git 仓库、分支、Jenkins Job、环境变量、部署配置等
3. 填充后用户仍可修改任何字段

## 五、前后端接口对照

### 创建流水线

| 维度 | 说明 |
|------|------|
| **前端 API** | `src/api/platform/pipeline.js` → `createPipeline(data)` |
| **HTTP 请求** | `POST /api/v1/k8s/cicd/pipeline/create` |
| **后端 Controller** | `internal/app/controllers/api/v1/cicd/pipeline_controller.go` → `Create()` |
| **后端 Request** | `internal/app/requests/cicd_pipeline.go` → `PipelineCreateRequest` |
| **后端 Service** | `internal/app/services/cicd_pipeline.go` → `PipelineCreate()` |

### 请求体结构（JSON）

```json
{
  "name": "my-java-pipeline",
  "description": "Java Spring Boot 项目流水线",
  "git_repo": "https://github.com/org/my-app.git",
  "git_branch": "main",
  "jenkins_url": "",
  "jenkins_job": "k8s-builder-java",
  "language_type": "java",
  "env_vars": [
    { "name": "JAVA_VERSION", "value": "17" }
  ],
  "deploy_config": {
    "replicas": 3,
    "strategy": "rollingUpdate",
    "resources": {
      "limits": { "cpu": "500m", "memory": "512Mi" },
      "requests": { "cpu": "200m", "memory": "256Mi" }
    }
  },
  "auto_deploy": true,
  "target_cluster_id": 1,
  "target_namespace": "production",
  "target_workload_kind": "Deployment",
  "target_workload_name": "my-app",
  "target_container": "",
  "deploy_env": "prod",
  "require_approval": true
}
```

### 后端验证规则

| 字段 | 规则 | 错误提示 |
|------|------|----------|
| `name` | `required`, `between:1,100` | 流水线名称不能为空 / 长度应在1-100之间 |
| `git_repo` | `required`, `url` | Git仓库地址不能为空 / 格式无效 |
| `language_type` | `in:go,java,frontend,python,custom` | 语言类型无效 |
| `jenkins_job` | 非必填（模板化自动推导） | - |

## 六、关联 API 汇总

| API | 方法 | 说明 | 触发时机 |
|-----|------|------|----------|
| `/api/v1/k8s/cicd/pipeline/create` | POST | 创建流水线 | 最终提交 |
| `/api/v1/k8s/cicd/pipeline/update` | POST | 更新流水线 | 编辑模式提交 |
| `/api/v1/k8s/cicd/template/list` | GET | 获取模板列表 | 页面初始化 |
| `/api/v1/k8s/cicd/git/branches` | POST | 获取 Git 分支列表 | 点击"获取分支" |
| `/api/v1/k8s/cicd/resource/templates` | GET | 获取资源模板 | 步骤4加载 |
| `/api/v1/k8s/cicd/resource/validate` | POST | 资源配置校验 | 选择资源模板后 |
| `/api/v1/k8s/cluster/list` | GET | 获取集群列表 | 步骤5加载 |
| `namespace/list` | GET | 获取命名空间列表 | 选择集群后 |
| `deployment/list` 等 | GET | 获取工作负载列表 | 选择命名空间后 |

## 七、表单数据结构（前端完整定义）

```javascript
// PipelineCreate.vue → pipelineData
{
  name: '',                    // 流水线名称
  description: '',             // 描述
  git_repo: '',                // Git 仓库地址
  git_branch: 'main',         // Git 分支（默认 main）
  jenkins_url: '',             // Jenkins 服务器地址（可空）
  jenkins_job: '',             // Jenkins Job 名称
  env_vars: [],                // 环境变量 [{name, value}]
  deploy_config: {             // 部署配置
    replicas: 3,               // 副本数（默认 3）
    strategy: 'rollingUpdate', // 部署策略（默认滚动更新）
    resources: {
      limits:   { cpu: '500m', memory: '512Mi' },
      requests: { cpu: '200m', memory: '256Mi' }
    }
  },
  auto_deploy: false,          // 自动部署开关（默认关闭）
  target_cluster_id: 0,        // 目标集群 ID
  target_namespace: '',        // 目标命名空间
  target_workload_kind: 'Deployment', // 工作负载类型
  target_workload_name: '',    // 工作负载名称
  target_container: '',        // 容器名称
  deploy_env: 'dev',           // 部署环境（默认开发）
  require_approval: true       // 部署前审批（默认开启）
}
```

| 步骤 | 字段 | 验证逻辑 |
|------|------|----------|

## 十、常见操作流程示例

### 示例：创建 Java Spring Boot 项目流水线

**步骤 1 - 基本信息：**
- 流水线名称：`user-service-pipeline`
- 描述：`用户服务 Spring Boot 项目自动构建部署`

**步骤 2 - 代码仓库：**
- Git 仓库地址：`https://github.com/myorg/user-service.git`
- 点击「获取分支」→ 选择 `main`

**步骤 3 - Jenkins 配置：**
- Jenkins 服务器地址：留空（使用系统默认）
- Jenkins Job 名称：`k8s-builder-java`（或使用快速模板自动填充）
- 添加环境变量：`JAVA_VERSION=17`

**步骤 4 - 部署策略：**
- 部署环境：生产环境
- 服务类型：Java
- 部署策略：滚动更新
- 副本数：3
- 资源配置：CPU limits 1000m / Memory limits 1Gi

**步骤 5 - 自动部署：**
- 启用自动部署：开
- 部署前审批：开（生产环境建议开启）
- 目标集群：production-cluster
- 目标命名空间：user-service-ns
- 工作负载类型：Deployment
- 工作负载名称：user-service
- 容器名称：留空（更新第一个容器）

**点击「创建流水线」→ 成功后跳转到流水线列表页**
