# K8sOperation 平台功能详细说明文档

> 本文档详细说明平台各功能模块的作用和实现，用于面试讲解和项目介绍。

---

## 一、平台定位

**K8sOperation** 是一个 **多集群 Kubernetes 权限管控平台**，参考 Rancher、KubeSphere、Kuboard 等企业级平台设计，提供：

- 多集群统一管理
- 细粒度 RBAC 权限控制
- CI/CD 流水线（含人工审批）
- 镜像仓库管理
- 资源全生命周期管理

## 三、核心功能模块详解

### 3.1 认证与用户管理

| 文件 | 功能 |
|------|------|
| `services/auth.go` | 用户登录认证 |
| `services/user.go` | 用户增删改查 |
| `models/user.go` | 用户数据模型 |
| `middlewares/auth.go` | JWT 认证中间件 |

**功能说明**：
- JWT Token 认证
- 用户登录/登出
- 密码加密存储
- Session 管理

---

### 3.2 RBAC 权限中心

| 文件 | 功能 |
|------|------|
| `models/sys_rbac.go` | 角色/权限数据模型 |
| `services/rbac.go` | 权限业务逻辑 |
| `routers/rbac/` | 权限管理路由 |
| `routers/k8srbac/` | K8s RBAC 路由 |

**角色类型**：

| 角色 | 权限范围 |
|------|----------|
| `super_admin` | 超级管理员，全平台权限 |
| `cluster_admin` | 集群管理员，特定集群完整权限 |
| `developer` | 开发者，指定命名空间操作权限 |
| `viewer` | 只读用户，仅查看权限 |

**权限操作**：

| 操作 | 说明 |
|------|------|
| `view` | 查看资源 |
| `create` | 创建资源 |
| `update` | 更新资源 |
| `delete` | 删除资源 |
| `exec` | 执行（如进入容器终端） |
| `manage` | 完整管理权限 |

**集群级权限控制**（`sys_user_cluster` 表）：
- 用户可分配到特定集群
- 可限制访问的命名空间列表
- 权限可设置过期时间

---

### 3.3 多集群管理

| 文件 | 功能 |
|------|------|
| `models/k8s_cluster.go` | 集群数据模型 |
| `services/k8s_cluster.go` | 集群业务逻辑 |
| `services/k8s_cluster_factory.go` | 集群客户端工厂 |
| `routers/kube_cluster/` | 集群管理路由 |

**功能说明**：

| 功能 | 说明 |
|------|------|
| 集群注册 | 上传 KubeConfig 接入集群 |
| KubeConfig 加密 | AES-256 加密存储，安全可靠 |
| 健康检测 | 定期检测集群连通性 |
| 版本获取 | 自动识别 K8s 版本 |
| 多集群切换 | 前端一键切换操作集群 |

**安全设计**：
```go
// KubeConfig 加密存储
func (k *K8sCluster) SetKubeConfig(plaintext string) error {
    encrypted, err := utils.GlobalEncryptKubeConfig(plaintext)
    k.KubeConfig = encrypted  // DB 中存加密数据
    return err
}
```

### 3.4 K8s 资源管理

平台支持管理以下 K8s 资源：

#### 工作负载类

| 资源 | 路由目录 | 服务文件 | 功能 |
|------|----------|----------|------|
| Deployment | `routers/kube_deployment/` | `services/k8s_deployment.go` | 无状态应用部署 |
| StatefulSet | `routers/kube_statefulset/` | `services/k8s_statefulset.go` | 有状态应用部署 |
| DaemonSet | `routers/kube_daemonset/` | `services/k8s_daemonset.go` | 守护进程部署 |
| Job | `routers/kube_job/` | `services/kube_job.go` | 一次性任务 |
| CronJob | `routers/kube_cronjob/` | `services/kube_cronjob.go` | 定时任务 |
| Pod | `routers/kube_pod/` | `services/k8s_pod.go` | Pod 管理/日志/终端 |

#### 网络类

| 资源 | 路由目录 | 功能 |
|------|----------|------|
| Service | `routers/kube_service/` | 服务发现与负载均衡 |
| Ingress | `routers/kube_ingress/` | 七层路由/域名配置 |

#### 配置与存储类

| 资源 | 路由目录 | 功能 |
|------|----------|------|
| ConfigMap | `routers/kube_configmap/` | 配置管理 |
| Secret | `routers/kube_secret/` | 敏感信息管理 |
| PV | `routers/kube_pv/` | 持久卷 |
| PVC | `routers/kube_pvc/` | 持久卷声明 |
| StorageClass | `routers/kube_storageclass/` | 存储类 |

#### 集群类

| 资源 | 路由目录 | 功能 |
|------|----------|------|
| Namespace | `routers/kube_namespace/` | 命名空间管理 |
| Node | `routers/kube_node/` | 节点管理/调度 |
| CRD | `routers/kube_crd/` | 自定义资源定义 |

**K8s 资源操作封装**（`pkg/k8s/` 目录）：
- 每种资源有独立的包（如 `pkg/k8s/deployment/`）
- 统一的 CRUD 接口
- 支持列表筛选、分页、排序
- 支持 YAML 导入/导出

### 3.5 CI/CD 流水线

| 文件 | 功能 |
|------|------|
| `models/cicd_pipeline.go` | 流水线数据模型 |
| `services/cicd_pipeline.go` | 流水线核心逻辑 |
| `services/cicd_stage.go` | 阶段管理（含审批） |
| `services/cicd_notify.go` | 钉钉通知 |
| `services/cicd_executor.go` | 部署执行器 |
| `routers/kube_cicd/` | CICD 路由 |

**流水线阶段**：

| 阶段 | 类型 | 执行方 |
|------|------|--------|
| 代码检出 | `checkout` | Jenkins |
| 构建 | `build` | Jenkins |
| 测试 | `test` | Jenkins |
| 推送镜像 | `push` | Jenkins |
| **人工审批** | `approval` | **平台** |
| 部署 | `deploy` | **平台** |

**人工审批机制**：
- 生产环境强制审批
- 记录审批人、时间、意见
- 审批通过后自动触发部署
- 钉钉实时通知

**详细架构见**：`docs/CICD_平台架构设计文档.md`

## 七、面试亮点总结

| 特性 | 实现 | 价值 |
|------|------|------|
| 多集群管理 | KubeConfig 加密存储，工厂模式创建客户端 | 安全、可扩展 |
| 细粒度 RBAC | 用户-角色-权限-集群四层模型 | 满足企业级权限需求 |
| CI/CD 人工审批 | 审批阶段独立记录，支持通过/拒绝 | 生产环境安全发布 |
| 平台健康监控 | 并发检测多集群，Rancher 风格仪表板 | 运维可视化 |
| 镜像仓库集成 | 支持多种仓库类型，统一接口 | 兼容性强 |
| 通知闭环 | 钉钉实时推送构建/审批/部署状态 | 团队协作高效 |
