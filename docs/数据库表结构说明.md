# K8sOperation 数据库表结构说明

> 本文档详细说明 `k8s-platform` 数据库中各表的用途，共 31 张表/视图。

---

## 一、数据库概览

| 分类 | 表数量 | 说明 |
|------|--------|------|
| 用户与认证 | 2 | 用户管理、审计日志 |
| RBAC 权限 | 5 | 角色、权限、关联关系 |
| IAM 项目权限 | 9 | 项目、用户组、授权（含2个视图） |
| K8s 集群 | 1 | 集群配置管理 |
| CI/CD 流水线 | 10 | 流水线、运行记录、阶段、审批 |
| 镜像仓库 | 3 | 仓库配置、清理策略 |
| 平台配置 | 1 | 全局设置 |
| **合计** | **31** | |

---

## 二、表结构详解

### 2.1 用户与认证（2 张）

| 表名 | 用途 | 核心字段 |
|------|------|----------|
| `user` | 用户基础信息 | id, username, password, email, phone, avatar, status |
| `audit_log` | 操作审计日志 | id, user_id, action, resource, ip, created_at |

---

### 2.2 RBAC 权限（5 张）

| 表名 | 用途 | 核心字段 |
|------|------|----------|
| `sys_role` | 系统角色定义 | id, name, display_name, role_type, is_system |
| `sys_permission` | 系统权限定义 | id, name, resource_type, action, parent_id |
| `sys_role_permission` | 角色-权限关联 | id, role_id, permission_id |
| `sys_user_role` | 用户-角色关联 | id, user_id, role_id |
| `sys_user_cluster` | 用户-集群权限 | id, user_id, cluster_id, role_type, namespaces, can_view, can_create, can_update, can_delete, can_exec |

**角色类型说明**：
```
super_admin   - 超级管理员，全平台权限
cluster_admin - 集群管理员，特定集群完整权限
developer     - 开发者，指定命名空间操作权限
viewer        - 只读用户，仅查看权限
```

---

### 2.3 IAM 项目权限（9 张，含 2 个视图）

| 表名 | 用途 | 核心字段 |
|------|------|----------|
| `iam_project` | 项目管理 | id, name, description, owner_id |
| `iam_project_member` | 项目成员 | id, project_id, user_id, role |
| `iam_group` | 用户组 | id, name, description |
| `iam_group_user` | 用户组成员 | id, group_id, user_id |
| `iam_role_template` | 角色模板 | id, name, permissions |
| `iam_grant` | 授权记录 | id, subject_type, subject_id, resource_type, resource_id |
| `iam_env_binding` | 环境绑定 | id, project_id, env_name, cluster_id, namespace |
| `iam_env_audit_log` | 环境审计日志 | id, env_id, user_id, action |
| `v_user_env_permissions` | 【视图】用户环境权限 | - |
| `v_user_permissions` | 【视图】用户权限汇总 | - |

---

### 2.4 K8s 集群（1 张）

| 表名 | 用途 | 核心字段 |
|------|------|----------|
| `kube_cluster` | K8s 集群配置 | id, cluster_name, cluster_version, kube_config(加密), status, last_check_at, last_error |

**状态说明**：
```
0 - OK（正常）
1 - BAD（异常）
2 - PENDING（未检测）
```

---

### 2.5 CI/CD 流水线（10 张）

| 表名 | 用途 | 核心字段 |
|------|------|----------|
| `cicd_pipeline` | 流水线定义 | id, name, git_repo, git_branch, jenkins_url, jenkins_job, target_cluster_id, target_namespace, target_workload_name, require_approval, auto_deploy |
| `cicd_pipeline_run` | 流水线运行记录 | id, pipeline_id, build_number, status, git_commit, image_url, image_digest, duration_sec |
| `cicd_pipeline_stage` | 流水线阶段记录 | id, run_id, stage_type, stage_name, status, approval_user_id, approval_decision, deploy_cluster_id, deploy_image |
| `cicd_approval` | 审批记录 | id, pipeline_id, env_name, status, request_user_id, approval_user_id, approval_comment |
| `cicd_environment` | 部署环境配置 | id, name, cluster_id, namespace, require_approval |
| `cicd_build` | 构建记录 | id, pipeline_id, build_number, status, image |
| `cicd_release` | 发布单 | id, app_name, namespace, workload_kind, workload_name, status, concurrency |
| `cicd_release_stage` | 发布阶段 | id, release_id, stage_type, status |
| `cicd_release_task` | 发布任务 | id, release_id, cluster_id, target_image, prev_image, status |

**阶段类型说明**：
```
checkout - 代码检出
build    - 构建
test     - 测试
push     - 推送镜像
approval - 人工审批
deploy   - 部署
```

**阶段状态说明**：
```
pending  - 等待中
running  - 执行中
success  - 成功
failed   - 失败
skipped  - 跳过
waiting  - 等待审批
aborted  - 已中止
```

---

### 2.6 镜像仓库（3 张）

| 表名 | 用途 | 核心字段 |
|------|------|----------|
| `image_registry` | 镜像仓库配置 | id, name, type, url, username, password(加密), access_key_id, access_key_secret(加密), region, insecure, is_default, status |
| `image_cleanup_policy` | 镜像清理策略 | id, registry_id, repository, keep_count, keep_days, enabled |
| `image_cleanup_log` | 镜像清理日志 | id, policy_id, deleted_tags, deleted_count, status |

**仓库类型说明**：
```
docker - Docker Registry V2
harbor - Harbor 私有仓库
acr    - 阿里云容器镜像服务
gcr    - Google Container Registry
ecr    - AWS ECR
quay   - Quay.io
```

---

### 2.7 平台配置（1 张）

| 表名 | 用途 | 核心字段 |
|------|------|----------|
| `platform_settings` | 平台全局设置 | id, key, value, description, category |

## 四、核心表 DDL 示例

### 4.1 用户表

```sql
CREATE TABLE `user` (
    `id` int unsigned NOT NULL AUTO_INCREMENT,
    `username` varchar(50) NOT NULL,
    `password` varchar(255) NOT NULL,
    `email` varchar(100) DEFAULT '',
    `phone` varchar(20) DEFAULT '',
    `avatar` varchar(255) DEFAULT '',
    `status` tinyint unsigned DEFAULT 1,
    `created_at` int unsigned DEFAULT 0,
    `modified_at` int unsigned DEFAULT 0,
    PRIMARY KEY (`id`),
    UNIQUE KEY `idx_username` (`username`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 4.2 集群表

```sql
CREATE TABLE `kube_cluster` (
    `id` int unsigned NOT NULL AUTO_INCREMENT,
    `cluster_name` varchar(100) NOT NULL,
    `cluster_version` varchar(50) DEFAULT '',
    `kube_config` text NOT NULL COMMENT 'AES加密存储',
    `status` tinyint unsigned DEFAULT 2 COMMENT '0=OK,1=BAD,2=PENDING',
    `last_check_at` bigint unsigned DEFAULT 0,
    `last_error` varchar(500) DEFAULT '',
    `created_at` bigint unsigned DEFAULT 0,
    `modified_at` bigint unsigned DEFAULT 0,
    `deleted_at` bigint unsigned DEFAULT 0,
    `is_del` tinyint unsigned DEFAULT 0,
    PRIMARY KEY (`id`),
    UNIQUE KEY `idx_cluster_name` (`cluster_name`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 4.3 流水线阶段表

```sql
CREATE TABLE `cicd_pipeline_stage` (
    `id` bigint NOT NULL AUTO_INCREMENT,
    `run_id` bigint NOT NULL,
    `pipeline_id` bigint NOT NULL,
    `stage_order` int DEFAULT 0,
    `stage_type` varchar(32) NOT NULL COMMENT 'checkout/build/test/push/approval/deploy',
    `stage_name` varchar(100) NOT NULL,
    `status` varchar(32) DEFAULT 'pending',
    `started_at` bigint unsigned DEFAULT NULL,
    `finished_at` bigint unsigned DEFAULT NULL,
    `duration_sec` int DEFAULT 0,
    `logs` longtext,
    `approval_user_id` bigint DEFAULT NULL,
    `approval_comment` varchar(500) DEFAULT '',
    `approval_decision` varchar(20) DEFAULT '',
    `deploy_cluster_id` bigint DEFAULT NULL,
    `deploy_namespace` varchar(100) DEFAULT '',
    `deploy_workload_kind` varchar(50) DEFAULT '',
    `deploy_workload_name` varchar(200) DEFAULT '',
    `deploy_container` varchar(100) DEFAULT '',
    `deploy_image` varchar(500) DEFAULT '',
    `error_message` varchar(1000) DEFAULT '',
    `created_at` bigint unsigned NOT NULL,
    `modified_at` bigint unsigned NOT NULL,
    PRIMARY KEY (`id`),
    KEY `idx_run_id` (`run_id`),
    KEY `idx_pipeline_id` (`pipeline_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```
